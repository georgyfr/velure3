================================================================================
  3. MATRICE DE DEPENDANCE ENTRE COMPOSANTS
================================================================================

Ce chapitre identifie les relations de dependance entre les composants
internes du theme NutriVitaX Pro. Une fleche ---> signifie "depend de"
(c'est-a-dire que le composant source ne peut pas fonctionner sans le
composant cible).

3.1 DEPENDANCES CRITIQUES (Chaines de dependance)

  CHAINE 1 : ARCHITECTURE THEME
    [C] functions.php
         ---> [C] setup.php
         ---> [C] theme.json
         ---> [C] style.css (design tokens)
         ---> [FE-01] React 18 (via @wordpress/scripts)
         ---> [CORE-01] WordPress 6.7+
         ---> [CORE-06] Gutenberg Editor

  CHAINE 2 : WOOCOMMERCE CORE
    [C] woo-enhancements.php
         ---> [CORE-02] WooCommerce 9.x
         ---> [WOO-03] HPOS (High Performance Order Storage)
         ---> [WOO-01] Cart Blocks
         ---> [WOO-02] Checkout Blocks
         ---> [CORE-04] MariaDB (tables wp_posts, wp_woocommerce*)

  CHAINE 3 : IA NUTRITIONNELLE
    [C] ai-recommendations.php
         ---> [AI-01] API Claude (Anthropic)
         ---> [D] nvx_health_profiles (table custom)
         ---> [D] nvx_ai_recommendations (table custom)
         ---> [C] quiz-engine.php
         ---> [D] nvx_quiz_responses (table custom)
         ---> [CORE-02] WooCommerce (produits a recommander)

  CHAINE 4 : STACK BUILDER
    [C] stack-builder.php
         ---> [CORE-02] WooCommerce (produits)
         ---> [D] nvx_supplement_stacks (table custom)
         ---> [D] nvx_health_profiles (profil utilisateur)
         ---> [C] ai-recommendations.php (recommandations IA)
         ---> [FE-01] React 18 (drag-and-drop UI)
         ---> [WOO-06] WooCommerce Subscriptions (abonnement stack)

  CHAINE 5 : FIDELITE NUTRIPOINTS
    [C] loyalty-points (logique)
         ---> [D] nvx_loyalty_points (table custom)
         ---> [CORE-01] WordPress (user_id)
         ---> [CORE-02] WooCommerce (commandes, produits achetes)
         ---> [C] quiz-engine.php (points quiz)

  CHAINE 6 : TRAÇABILITE INGREDIENTS
    [C] ingredient-transparency (logique)
         ---> [D] nvx_ingredient_origins (table custom)
         ---> [GEO-01/02] Mapbox ou Leaflet (carte)
         ---> [C] woo-enhancements.php (meta boxes produit)

  CHAINE 7 : QUIZ SANTE
    [C] quiz-engine.php
         ---> [D] nvx_quiz_responses (table custom)
         ---> [D] nvx_health_profiles (profil resultant)
         ---> [C] ai-recommendations.php (recommandations)
              ---> [AI-01] API Claude (si IA active)
         ---> [C] templates/page-quiz.html (template FSE)
         ---> [FE-01] React 18 (UI interactive)

3.3 GRAPHES DE DEPENDANCE CIRCULAIRE (a eviter)

  ATTENTION : Certaines dependances risquent de creer des cycles :

  Cycle potentiel 1 :
    quiz-engine ---> ai-recommendations ---> nvx_health_profiles
    ---> quiz-engine (re-scoring apres quiz)

  Resolution : Le quiz-engine ecrit le profil PUIS appelle l'IA en
  callback (pas en dependance synchrone). L'IA lit le profil sans
  modifier les reponses du quiz.

  Cycle potentiel 2 :
    stack-builder ---> ai-recommendations ---> nvx_supplement_stacks
    ---> stack-builder (recommandation "produit manquant")

  Resolution : L'IA retourne des suggestions qui sont affichees dans
  le stack builder sans modifier le stack existant. L'utilisateur
  decide manuellement d'ajouter ou non.

  7. ANALYSE DES RISQUES D'INTEGRATION
================================================================================

7.1 RISQUES CRITIQUES (Bloquants potentiels)

  R01 | CONFLIT ELEMENTOR vs FSE
  Composants : [Elementor] vs [CORE-07] theme.json FSE
  Nature : Incompatibilite architecturale
  Probabilite : HAUTE (90%)
  Impact : Eleve — Elementor utilise son propre systeme de templates
            qui entre en conflit avec les templates FSE de WordPress 6.7+
  Mitigation :
    - Desactiver Elementor quand NutriVitaX Pro est actif
    - Ou utiliser Elementor uniquement pour les pages specifiques
      (pas les templates WooCommerce)
    - Tester la coexistence avec Elementor 4.x+ qui a un support
      FSE partiel

  R02 | API CLAUDE INDISPONIBLE
  Composants : [AI-01] API Claude
  Nature : Dependance externe unique (Single Point of Failure)
  Probabilite : MOYENNE (30%)
  Impact : Eleve — Toutes fonctionnalites IA bloquées sans l'API
  Mitigation :
    - Implementer un systeme rule-based complet en fallback (Phase 1)
    - Architecture multi-provider : Claude -> OpenAI -> IA locale
    - Cache des recommandations IA dans nvx_ai_recommendations
      (ne pas rappeler l'API pour le meme profil)

  R03 | WOOCOMMERCE SUBSCRIPTIONS NON DISPONIBLE
  Composants : [WOO-06] WooCommerce Subscriptions (~279$/an)
  Nature : Licence payante non acquise
  Probabilite : MOYENNE (40%)
  Impact : Moyen — Abonnements bloqués, mais stack builder peut
            fonctionner sans
  Mitigation :
    - Developper un systeme d'abonnement simplifie custom en PHP
    - Ou utiliser WooCommerce Subscriptions en mode trial
    - Lier l'achat du theme a une licence incluse si possible

7.2 RISQUES MODERES

  R04 | PERFORMANCE SANS REDIS
  Composants : [PERF-02] Redis, [PERF-03] Object Cache Pro
  Nature : Infrastructure non disponible en dev local
  Probabilite : HAUTE en dev (100%), FAIBLE en prod (20%)
  Impact : Moyen — Objectifs Core Web Vitals non atteignables en
            environnement de developpement
  Mitigation : Utiliser transient API WordPress + OPcache comme
            substitut. Les objectifs extremes sont des cibles production.

  R05 | GOOGLE PLACES API PAYANTE
  Composants : [MED-03] Google Places API (adresse auto-complete)
  Nature : Cout par requete
  Probabilite : FAIBLE
  Impact : Faible — fonctionnalite optionnelle checkout
  Mitigation : Implementer un autocomplete base sur la base de donnees
            locale (codes postaux + villes) comme alternative gratuite.

  R06 | MAPBOX TROP CHER
  Composants : [GEO-01] Mapbox GL JS
  Nature : Cout par utilisation (map loads)
  Probabilite : MOYENNE (50%)
  Impact : Faible — carte interactive ingredients
  Mitigation : Utiliser [GEO-02] Leaflet + OpenStreetMap (100% gratuit)
            comme alternative immediate. Mapbox reste optionnel.

7.3 RISQUES FAIBLES

  R07 | RETARD DEPENDANCE NPM
  Composants : [FE-01] React 18, [FE-03] Vite 6, [FE-04] @wordpress/scripts
  Nature : Probleme d'installation en user-space
  Probabilite : FAIBLE (15%)
  Impact : Faible — Build JS bloque
  Mitigation : Installer nvm en user-space, version pinnee dans
            package.json

  R08 | CHANGEMENT D'API CLAUDE
  Composants : [AI-01] API Claude
  Nature : Breaking changes sur l'API
  Probabilite : FAIBLE (20%)
  Impact : Moyen — IA ne fonctionne plus temporairement
  Mitigation : Versionner les appels API, implementer un proxy
            WordPress qui fait abstraction du provider.

  R09 | WPML INCOMPATIBLE FSE
  Composants : [I18N-01] WPML
  Nature : WPML peut ne pas supporter pleinement les templates FSE
  Probabilite : MOYENNE (40%)
  Impact : Faible — Traductions manuelles necessaires
  Mitigation : Utiliser le systeme de traduction natif WordPress
            (textdomain) en priorite. WPML comme couche superieure.
